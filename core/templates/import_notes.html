Each row should contain only one product. If a company has multiple products, create separate rows with the same company details and different product-related information.
2
Leave a cell blank if you do not have a value for that field.
3
For multiple phone numbers, fax numbers, or email addresses, separate values using a , (comma) only. Do not use semicolons ; or any other separator — the import will fail.
4
Do not insert commas within a single value (e.g. a single phone number). Commas are treated as separators between multiple entries.
5
If a company spans multiple rows, all company-level fields must be identical across every row. Only the product-related fields should differ.
6
For multiple addresses, use columns Address1 / City1 / State1 / Country1 for the first address, Address2 / City2 / State2 / Country2 for the second, and so on.
7
Verify that all information is accurate and properly formatted before importing. Date fields must be in DD-MM-YYYY or YYYY-MM-DD format.
8
If any field contains invalid data, the company name and field(s) with errors will be shown after import. Correct the data and upload the file again.
9
After a successful import, records can be filtered and edited from within the application.
10
Do not modify column headers. The import process relies on exact header names as provided in the template.
